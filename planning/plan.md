Обновлён: 2026-06-23

## Сделано
- Создан репозиторий, добавлен `.gitignore`

- Контроллер игрока (ветка `feature/player-controller`): `CharacterBody3D`, обзор мышью (yaw крутит тело, pitch — узел `Head` с камерой, клампится), движение WASD в дум-стиле (резкий разгон/торможение через `move_toward`, малая инерция, `air_control` в воздухе), прыжок + ручная гравитация, **буфер прыжка + койот-тайм**. Захват мыши по клику / `ui_cancel` (Esc) — освобождение (под веб/pointer lock). Все параметры — в `@export`. Скрипты: `res://scenes/player/player.gd`, `res://scenes/main/main.gd`.
- Единая точка входа `res://scenes/main/main.tscn` (`main.gd`): грузит уровень из `@export level_scene` и спавнит игрока (`@export player_scene`) в `Marker3D` с именем `PlayerSpawn`.
- Тестовый уровень `res://scenes/levels/test_level.tscn` — временная «коробка» на CSG (пол + препятствия + спавн), для отладки. Боевой уровень — позже (Этап 3).
- Input Map: `move_forward/back/left/right` (WASD), `jump` (Space), `shoot` (ЛКМ).
- **Стрельба** (ветка `feature/shooting`): hitscan-оружие, semi-auto с кулдауном. Оружие — экранный спрайт-оверлей (`Control` в `CanvasLayer`, screen-space), пока рисованный плейсхолдер через `_draw` (серый ствол + вспышка). Прицел-крестик — отдельный `Control`, рисуется кодом. Hitscan — луч из активной камеры (`get_camera_3d()` → `intersect_ray`), тело игрока исключено. При попадании вызывается `take_damage(amount)` у цели, если метод есть. Захватывающий клик поглощается в `player._input`, выстрел — только при захваченном курсоре. Звук — опциональный `@export shot_sound: AudioStream` (плейсхолдер молчит, пока ассет не назначен). Скрипты: `res://scenes/player/weapon.gd`, `res://scenes/ui/crosshair.gd`; правка `res://scenes/player/player.gd` (захват курсора перенесён в `_input`).
- **Враг** (ветка `feature/enemy`): наземный враг ближнего боя, спрайт-биллборд, базовый ИИ, здоровье, смерть.
  - `HealthComponent` (`res://scripts/health_component.gd`, `class_name HealthComponent extends Node`) — переиспользуемый компонент здоровья: `max_health`/`current_health` (read-only property), `take_damage`, `is_alive`, кламп; сигналы `health_changed(current, max)` и `died`. Один и тот же для врага и (позже) игрока — реакция на события снаружи через сигналы.
  - `EnemyBase` (`res://scenes/enemies/enemy_base.gd`, `class_name EnemyBase extends CharacterBody3D`) — играбелен «как есть» и служит базой. Машина состояний `IDLE → CHASE → ATTACK → DEAD` (enum + `match`). Все характеристики — `@export` (`move_speed`, `detection_range`, `attack_range`, `attack_damage`, `attack_cooldown`, `gravity`, `target_group`). Конвенция урона: `take_damage` на теле делегирует в `HealthComponent`; на `died` — `_on_death` (стоп ИИ, снять коллизию через `set_deferred`, «положить» спрайт, `queue_free` через 2 с). Хуки для переопределения наследниками: `_can_see_target`, `_move_towards_target`, `_perform_attack`, `_on_death`.
  - Сцены: `res://scenes/enemies/enemy_base.tscn` (`CharacterBody3D` `Enemy` + `CollisionShape3D` капсула r0.4/h1.8 @y0.9 + `Sprite3D` биллборд + `HealthComponent`); `res://scenes/enemies/melee_grunt.tscn` — унаследованная сцена (демонстрация расширения «только цифрами»).
  - Игрок добавлен в группу `player` (правка `res://scenes/player/player.tscn`) — враг находит его через `get_first_node_in_group`.
  - Тестовый инстанс `melee_grunt` добавлен в `res://scenes/levels/test_level.tscn`.
- **Здоровье игрока + HUD + смерть/рестарт** (ветка `feature/player-health-hud`) — закрыт Этап 1.
  - Игроку добавлен дочерний `HealthComponent` (`max_health = 100`); тело получило `take_damage(amount)` с делегированием в компонент — удары врага теперь проходят по-настоящему (раньше телеграфировались вхолостую). `player.gd` остаётся «движение + приём урона».
  - HUD (`res://scenes/ui/hud.gd`, `class_name Hud extends Control`) — экранный, рисуется кодом (`_draw`, дефолтный шрифт, ноль веса билда): полоска + число HP слева внизу (краснеет ниже `low_threshold`), патроны справа как «∞» (плейсхолдер до системы патронов, Этап 2). В `_ready` принудительный `set_anchors_and_offsets_preset(PRESET_FULL_RECT)` (иначе `size = 0` и ничего не рисуется) и `process_mode = ALWAYS`. Все размеры/цвета/кегли — `@export`.
  - Сцена `res://scenes/ui/hud.tscn`: корень `CanvasLayer` `HUD` (слой 1 — поверх слоя оружия), внутри `Control` `Root` со скриптом.

Рестарт по любой клавише/клику: HUD ловит ввод на паузе (`ALWAYS`) и шлёт `restart_requested` наверх → `main` снимает паузу и `reload_current_scene()`.
  - Скрипты: `res://scenes/ui/hud.gd` (новый); правки `res://scenes/main/main.gd` (поднятие HUD, связывание сигналов, пауза/оверлей/рестарт) и `res://scenes/player/player.gd` (`HealthComponent` + `take_damage`). Сцены: новая `hud.tscn`; правки `player.tscn` (узел `HealthComponent`, max 100) и `main.tscn` (`@export hud_scene`).
- **Патроны + пикапы** (ветка `feature/pickups-and-ammo`) — первая фича Этапа 2, «∞» на HUD заменён реальным счётчиком.

- **Арсенал: несколько оружий + переключение + дымок от попадания** (ветка `feature/arsenal-and-switching`) — вторая фича Этапа 2.
  - `Weapon` (`res://scenes/player/weapon.gd`, `class_name Weapon extends Control`) — бывший одиночный ствол стал базой: настраивается ЦИФРАМИ через `@export` (`damage`/`fire_mode`/`fire_cooldown`/`pellets`/`spread_degrees`/`ammo_type`/`ammo_per_shot`/`slot`/`uses_ammo`/размеры плейсхолдера). `fire_mode` — enum `SEMI`/`AUTO`.   - `WeaponManager` (`res://scenes/player/weapon_manager.gd`, `class_name WeaponManager extends Node`) — узел в `WeaponLayer`, владеет дочерними `Weapon`, раскладывает их по слотам (`@export slot` каждого), прячет/показывает активный, роутит ввод. Новый ствол = дубликат-сцена с другими цифрами; иной тип атаки = наследник с переопределённым `_fire`.
  - `AmmoComponent`: добавлен пул `shells`. Input Map: `weapon_1..weapon_5` (клавиши 1–5), `weapon_next`/`weapon_prev` (колёсико вверх/вниз).

Пул переиспользуемых `CPUParticles3D` (round-robin), разовая эмиссия, биллборд, мягкая текстура-пятно генерится кодом (ноль веса). `main` связывает `weapons.surface_hit → ImpactSmoke.spawn` (signal up). - **Ракурсные спрайты врага + анимация** (ветка `feature/enemy-directional-sprites`) — третья фича Этапа 2. Думовские 8 ракурсов из 5 рисуемых (3 — зеркало `flip_h`) + покадровая анимация.
  - `SpriteAnimation` (`res://scripts/sprite_animation.gd`, `class_name SpriteAnimation extends Resource`) — конфиг одной анимации (стиль ScriptableObject, как `AmmoType`): `name`/`row_start`/`frame_count`/`fps`/`loop`/`directional`. Массив в инспекторе `DirectionalSprite3D`.
  - `DirectionalSprite3D` (`res://scripts/directional_sprite_3d.gd`, `class_name DirectionalSprite3D extends Sprite3D`) — скрипт прямо на узле `Sprite3D` врага. Нет текстуры/анимаций — сам генерит код-плейсхолдер и работает статикой (играбельно до арта). Сигнал `animation_finished`. Метод `flash_pain()` — вспышка `modulate` через `Tween` (реакция на урон). ## Решения по архитектуре
- 2026-06-23: **Снаряд — общая сущность `Projectile` (база + данные), реюз игроком и врагом.** И ракета, и файрбол — «летит → урон по удару → взрыв». По правилу двух вынесено в одну сущность; различие — только `@export`, носители `rocket.tscn`/`fireball.tscn`. Движение — заметающий луч (как хитскан в `weapon.gd`), не Area3D/физ-тело: без туннелирования быстрого снаряда, без GPU-частиц — безопасно для Compatibility/WebGL. Спавн — статическая фабрика `launch()`; снаряды в контейнере `world_projectiles`, не в стрелке (переживают его смерть). Эффект взрыва и звук — по группе (`explosions`/`combat_audio`), как `CombatAudio`, без проводки сигналов каждого снаряда.
- 2026-06-23: **AoE — `intersect_shape` сферой, без фракций.** Сплэш бьёт ВСЕХ в радиусе с `take_damage` (враги, стрелок, игрок), затухание по расстоянию — снаряд не различает своих/чужих (фича: задел под инфайтинг и рокет-джамп). В полёте исключается лишь стрелок (его RID), чтобы не детонировать у дула. Прямое попадание — при `splash_radius = 0` (файрбол).
- 2026-06-23: **Ракетомёт — наследник `Weapon` с переопределённым `_fire` (база + хук).** Ось «иной тип атаки = хук `_fire`»: вместо хитскана — `Projectile.launch`. Кулдаун/патроны/звук (`play_weapon` по слоту) — через базу. Урон/сплэш/отброс — на `rocket.tscn` (данные снаряда), не на оружии.
- 2026-06-23: **Отброс взрывом — конвенция `apply_knockback`, импульс ДО урона.** Зеркало `take_damage` (снаряд зовёт по `has_method`). Импульс перед уроном → убитый этим взрывом успевает «получить запуск» и улетает трупом (`_process_launch`: баллистика + оседание на пол). Живой отлетает смещением поверх ИИ (`move_and_collide`, т.к. ИИ перезадаёт velocity). Уже лежащие трупы не толкаются: на смерти `collision_layer = 0` → shape-запрос их не видит. Изменено: на смерти `collision_mask` больше НЕ обнуляется (запущенный труп сталкивается с полом; layer 0 оставляет его «сквозным» для пуль/ИИ).
