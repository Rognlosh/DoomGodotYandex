# План проекта — DOOM-like для Яндекс Игр
Обновлён: 2026-06-17
Движок: Godot 4.6.3 (GDScript) · Подход: low-poly 3D со стилизацией под ретро · Платформа: Яндекс Игры (HTML5)

## Сейчас в работе
- Активной фичи в работе нет.
- Ветка `feature/enemy` готова к мержу в `main` (код + обновлённый план) — мерж делает разработчик.
- Следующая фича (кандидат по бэклогу, Этап 1): **HUD** — здоровье и патроны. Подписывается на сигналы (`health_changed` у `HealthComponent` игрока и будущий счётчик патронов оружия) по принципу «signal up». Логически идёт перед/вместе с «урон игроку»: игрок получает `HealthComponent` (тот же, что у врага) + реакцию на смерть (game over/рестарт), а HUD показывает его состояние.

## Бэклог (надо сделать)

### Этап 1 — играбельный вертикальный срез (ядро)
- HUD: здоровье и патроны
- Урон игроку, смерть и рестарт (игроку — тот же `HealthComponent`, что у врага; реакция на `died` — game over/рестарт)

### Этап 2 — наполнение боя
- Пикапы: аптечки, патроны, броня
- Несколько типов оружия + переключение
- 2–3 типа врагов (расширяем `EnemyBase`: цифры в инспекторе или переопределение хуков — стреляющий/летающий)
- Звук: выстрелы, попадания, эмбиент

### Этап 3 — структура игры
- Загрузка уровней / структура карты
- Меню: старт, рестарт, настройки
- Прогресс между уровнями

### Этап 4 — платформа Яндекс
- Инициализация Yandex Games SDK
- Облачные сейвы прогресса
- Реклама (rewarded / interstitial) — определить точки показа
- Лидерборды
- Мобильное управление (тач): виртуальный джойстик + кнопки
- Сборка/экспорт в HTML5, упаковка под Яндекс, контроль размера (~20–25 МБ)

## Сделано
- Создан репозиторий, добавлен `.gitignore`
- Бутстрап проекта (ветка `feature/project-bootstrap`): создан проект Godot 4.6.3 (`config/name=DoomGodotYandex`). Рендер `gl_compatibility`, в т.ч. в `.mobile`-профиле — настроено под веб (WebGL 2.0). В репозитории: `project.godot`, `.gitattributes`, `.editorconfig`, `icon.svg`.
- Контроллер игрока (ветка `feature/player-controller`): `CharacterBody3D`, обзор мышью (yaw крутит тело, pitch — узел `Head` с камерой, клампится), движение WASD в дум-стиле (резкий разгон/торможение через `move_toward`, малая инерция, `air_control` в воздухе), прыжок + ручная гравитация, **буфер прыжка + койот-тайм**. Захват мыши по клику / `ui_cancel` (Esc) — освобождение (под веб/pointer lock). Все параметры — в `@export`. Скрипты: `res://scenes/player/player.gd`, `res://scenes/main/main.gd`.
- Единая точка входа `res://scenes/main/main.tscn` (`main.gd`): грузит уровень из `@export level_scene` и спавнит игрока (`@export player_scene`) в `Marker3D` с именем `PlayerSpawn`.
- Тестовый уровень `res://scenes/levels/test_level.tscn` — временная «коробка» на CSG (пол + препятствия + спавн), для отладки. Боевой уровень — позже (Этап 3).
- Input Map: `move_forward/back/left/right` (WASD), `jump` (Space), `shoot` (ЛКМ).
- **Стрельба** (ветка `feature/shooting`): hitscan-оружие, semi-auto с кулдауном. Оружие — экранный спрайт-оверлей (`Control` в `CanvasLayer`, screen-space), пока рисованный плейсхолдер через `_draw` (серый ствол + вспышка). Прицел-крестик — отдельный `Control`, рисуется кодом. Hitscan — луч из активной камеры (`get_camera_3d()` → `intersect_ray`), тело игрока исключено. При попадании вызывается `take_damage(amount)` у цели, если метод есть. Захватывающий клик поглощается в `player._input`, выстрел — только при захваченном курсоре. Звук — опциональный `@export shot_sound: AudioStream` (плейсхолдер молчит, пока ассет не назначен). Скрипты: `res://scenes/player/weapon.gd`, `res://scenes/ui/crosshair.gd`; правка `res://scenes/player/player.gd` (захват курсора перенесён в `_input`).
- **Враг** (ветка `feature/enemy`): наземный враг ближнего боя, спрайт-биллборд, базовый ИИ, здоровье, смерть.
  - `HealthComponent` (`res://scripts/health_component.gd`, `class_name HealthComponent extends Node`) — переиспользуемый компонент здоровья: `max_health`/`current_health` (read-only property), `take_damage`, `is_alive`, кламп; сигналы `health_changed(current, max)` и `died`. Один и тот же для врага и (позже) игрока — реакция на события снаружи через сигналы.
  - `EnemyBase` (`res://scenes/enemies/enemy_base.gd`, `class_name EnemyBase extends CharacterBody3D`) — играбелен «как есть» и служит базой. Машина состояний `IDLE → CHASE → ATTACK → DEAD` (enum + `match`). Все характеристики — `@export` (`move_speed`, `detection_range`, `attack_range`, `attack_damage`, `attack_cooldown`, `gravity`, `target_group`). Конвенция урона: `take_damage` на теле делегирует в `HealthComponent`; на `died` — `_on_death` (стоп ИИ, снять коллизию через `set_deferred`, «положить» спрайт, `queue_free` через 2 с). Хуки для переопределения наследниками: `_can_see_target`, `_move_towards_target`, `_perform_attack`, `_on_death`.
  - Сцены: `res://scenes/enemies/enemy_base.tscn` (`CharacterBody3D` `Enemy` + `CollisionShape3D` капсула r0.4/h1.8 @y0.9 + `Sprite3D` биллборд + `HealthComponent`); `res://scenes/enemies/melee_grunt.tscn` — унаследованная сцена (демонстрация расширения «только цифрами»).
  - Игрок добавлен в группу `player` (правка `res://scenes/player/player.tscn`) — враг находит его через `get_first_node_in_group`.
  - Тестовый инстанс `melee_grunt` добавлен в `res://scenes/levels/test_level.tscn`.

## Решения по архитектуре
- 2026-06-17: **Конвенция урона** — у всех damageable-объектов метод `take_damage(amount: float)`. Источник урона (оружие/враг) не знает о классах целей: проверяет `has_method("take_damage")` и вызывает. Метод лежит на физическом теле (по нему попадает луч/контакт) и делегирует во вложенный `HealthComponent`.
- 2026-06-17: **`HealthComponent` универсален для врага и игрока.** Компонент знает только числа (`current/max`, кламп, `take_damage`, сигналы `health_changed`/`died`). Что значит «умереть»/«получить урон» — решает владелец, подписавшись на сигналы (signal up): враг → труп+`queue_free`, игрок → game over/рестарт+HUD. Броня, i-frames, оверхил, `heal` — добавляются аддитивно (опц. `@export` или отдельное звено перед HP), форка компонента не требуют. Файл: `res://scripts/health_component.gd`.
- 2026-06-17: **Расширяемость врагов — база + хуки + статы.** `EnemyBase` (`class_name`, `extends CharacterBody3D`) = готовый наземный враг ближнего боя и одновременно база. Новый тип врага: (1) другие цифры — унаследованная сцена со своими `@export`, ноль кода; (2) другое поведение — `extends EnemyBase` с переопределением хуков (`_move_towards_target` — летающий; `_perform_attack` — стреляющий; `_can_see_target` — конус/360°; `_on_death` — особая смерть). Типы врагов оформляем унаследованными сценами от `enemy_base.tscn`.
- 2026-06-17: **Враг — спрайт-биллборд + отдельный хитбокс.** Визуал `Sprite3D` (`Billboard = Y-Billboard`, `Texture Filter = Nearest`, `Alpha Cut = Discard` — жёсткие края и обход сортировки прозрачности WebGL2/Safari, `Pixel Size ≈ 0.04`). Коллизия — отдельный `CollisionShape3D` (капсула) на теле; по спрайту луч не ловится, стреляем по телу (слой 1, как стены — текущий hitscan попадает без правок оружия). Плейсхолдер-текстура генерируется кодом (`Image`/`ImageTexture`, ноль веса билда); появится арт — назначить `texture`, генерация сама отключится.
- 2026-06-17: **Обнаружение врага — дистанция + линия видимости.** В `detection_range` И прямой луч (`intersect_ray` с высоты «глаз», исключая себя) упёрся в саму цель или ни во что — иначе не «видит» сквозь стены. Атака — **ближний бой**: подошёл в `attack_range` → бьёт по кулдауну через `take_damage` (до фичи «урон игроку» телеграфируется вхолостую — у игрока ещё нет метода).
- 2026-06-17: **Поиск цели — через группу `player`** (`get_first_node_in_group`), без жёстких ссылок. Игрок добавлен в группу в `player.tscn`. Имя группы — `@export target_group` у врага.
- 2026-06-17: **Оружие — экранный спрайт (screen-space), а не 3D-модель под камерой.** Ствол и вспышка живут в `CanvasLayer` (`Control` + `_draw`/будущие `TextureRect`), как в DOOM. Причины: ретро-эстетика, бюджет билда, капризы WebGL/Safari (без лишних динамических 3D-источников света). Логику выстрела (hitscan/кулдаун/ввод) — на том же `Control`; камеру/мир берёт через `get_viewport().get_camera_3d()`.
- 2026-06-17: **Прицеливание сейчас — точный 3D-луч из камеры.** Дум-овский автоприцел вынесен в опциональные фичи (см. выше), делается с врагами или не делается вовсе.
- 2026-06-17: **Координация «захват курсора vs выстрел»** — захватывающий клик обрабатывается и поглощается в `player._input` (раньше `_unhandled_input`), оружие стреляет в `_unhandled_input` только при `MOUSE_MODE_CAPTURED`. Опора на конвейер ввода `_input` → GUI → `_unhandled_input`, а не на порядок обхода узлов.
- 2026-06-17: **Единая точка входа** — `res://scenes/main/main.tscn` (Project Settings → Run → Main Scene). Грузит текущий уровень и спавнит игрока; позже сюда — HUD, смена уровней, пауза, game-over. Уровни/меню меняются — точка входа одна.
- 2026-06-17: **Композиция вместо дублирования** — переиспользуемую логику оформляем отдельными узлами-компонентами (реализовано: `HealthComponent` для игрока и врагов), а не копипастой скриптов.
- 2026-06-17: **Связь узлов — «Call down, signal up»** (паттерн «Наблюдатель»). Прямые вызовы только вниз по дереву; наверх/вбок (UI, реакции) — через сигналы. Источник данных не знает о потребителях (напр. `health_changed` → HP-бар подписывается сам). Глобальную «шину событий» (autoload signal hub) вводим позже, по необходимости.
- 2026-06-17: **Автозагрузок пока нет.** `GameState` (здоровье/патроны) — вместе с HUD/уроном; обёртка Yandex SDK — Этап 4. Папка `autoload/` зарезервирована, пустая.
- 2026-06-17: **Дум-ощущение движения** — высокие `acceleration`/`friction` (резко, малая инерция), `gravity ~24` (не «лунная»), отзывчивый прыжок (буфер + койот). Всё тюнится через `@export`.
- 2026-06-17: Конвенция имён веток — `feature/<имя-фичи>` (без префикса `claude`).
- 2026-06-17: Настройки окна/растяжки под веб (`stretch/mode`, aspect) отложены до Этапа 4 (экспорт HTML5).
- 2026-06-16: Графика — настоящее low-poly 3D в Godot, ретро-стилизация (пиксельные текстуры, спрайтовые враги). Не «честный» рейкастер.
- 2026-06-16: Язык — GDScript. Веб-рендер — Compatibility + WebGL 2.0.
- **Структура папок (подтверждена):**
  - `res://scenes/` — сцены по подпапкам: `main/`, `player/`, `levels/`, `enemies/`, `ui/`. Скрипт лежит рядом со своей сценой.
  - `res://scripts/` — только для скриптов без собственной сцены (компоненты, утилиты).
  - `res://assets/` — текстуры, спрайты, звук, шрифты.
  - `res://autoload/` — синглтоны (пока пусто; GameState, обёртка Yandex SDK — позже).
