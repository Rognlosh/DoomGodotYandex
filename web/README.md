# Веб-сборка под Яндекс Игры (Этап 4)

Экспорт в HTML5 + интеграция Yandex Games SDK. Игра **desktop-only** (клавиатура+мышь,
без тача), при публикации помечается «только для ПК».

## Что уже в репозитории

| Файл | Роль |
|---|---|
| `autoload/yandex_sdk.gd` | Обёртка над SDK (autoload `YandexSDK`): сейвы, реклама, лидерборды, LoadingAPI/GameplayAPI. Вне веба — безопасный фолбэк. |
| `web/head_include.html` | JS-мост: подключает `sdk.js`, поднимает `window.GodotYSDK`. Вставляется в `<head>` при экспорте. |
| `export_presets.cfg` | Готовый Web-пресет «Web (Yandex)» с уже вшитым `head_include` и `thread_support=false`. |

Рендер уже `gl_compatibility` (WebGL 2.0) в `project.godot` — менять не нужно.

## Разовая подготовка редактора

1. **Шаблоны экспорта.** Editor → Управление шаблонами экспорта → скачать для 4.7,
   если ещё не стоят.
2. Открыть проект в Godot — пресет «Web (Yandex)» подхватится из `export_presets.cfg`.
   - Если Godot пожалуется/сбросит пресет, создай его вручную (Проект → Экспорт →
	 Добавить → Web) и вставь содержимое `web/head_include.html` целиком в поле
	 **HTML → Head Include**. Остальное — по таблице ниже.

### Ключевые настройки пресета (проверить в диалоге экспорта)

- **Renderer:** Compatibility (наследуется из проекта).
- **HTML → Head Include:** содержимое `web/head_include.html` (уже вшито в `.cfg`).
- **Threads → Thread Support:** **ВЫКЛ.** Яндекс не отдаёт заголовки COOP/COEP
  (cross-origin isolation), а многопоточный WebGL-билд без них не стартует. Однопоточный
  билд обязателен.
- **Head Include** тянет `/sdk.js` — относительный путь. Он резолвится и на хостинге
  Яндекса, и через dev-proxy (см. ниже). Открытие голого `index.html` из файловой
  системы SDK не поднимет — обёртка уйдёт в офлайн-фолбэк (это нормально).

## Сборка

Проект → Экспорт → «Web (Yandex)» → Export Project → `build/web/index.html`
(папка `build/` в `.gitignore`? нет — не коммить билд; при желании добавь в `.gitignore`).

## Тестирование БЕЗ модерации (dev-режим)

Модерация нужна ТОЛЬКО для публикации в каталог и реальных показов рекламы. Весь SDK
(сейвы, реклама-плейсхолдеры, лидерборды, авторизация) проверяется локально через
официальный прокси:

```bash
npx @yandex-games/sdk-dev-proxy -p build/web --app-id=<ID_ЧЕРНОВИКА>
```

- `-p build/web` — папка с собранным `index.html`.
- `--app-id` — id черновика игры из консоли разработчика (нужен, чтобы прокси
  отдавал корректный `/sdk.js`). Для чистого мока можно `--mock` (см. `--help`).
- Прокси поднимет `https://localhost:8080/...` — открывать оттуда, тогда `/sdk.js`
  подхватится и `YandexSDK.is_online()` станет true.

Замоканы: реклама (плейсхолдер-оверлеи), `player.setData/getData` (localStorage),
лидерборды, авторизация. Реальные деньги/показы — только после публикации.

## Что проверять

- **LoadingAPI:** «крутилка» Яндекса пропадает после появления главного меню
  (`YandexSDK.notify_game_ready()` в роутере).
- **Сейвы:** пройти уровень эпизода → выйти в меню → перезайти в тот же эпизод:
  старт с сохранённого уровня (резюм). Настройки (громкости/чувствительность) тоже
  переживают перезагрузку вкладки.
- **Реклама:** interstitial — на кнопке «Дальше» между уровнями; rewarded — пункт
  паузы «Пополнить запасы (реклама)» (виден только онлайн) → после просмотра полное
  HP/патроны.
- **Лидерборд:** главное меню → «Рекорды» (таблица из мока); очки уходят на концовке
  эпизода.
- **GameplayAPI:** пауза/меню шлют `stop`, чистый геймплей — `start` (видно в
  сетевых запросах прокси/метриках).

## Лидерборд в консоли

Метод шлёт очки в лидерборд с техническим именем **`progress`** (метрика — суммарно
пройдено уровней, больше = выше). Перед публикацией создать лидерборд с этим именем
в консоли разработчика (Тип: число, порядок по убыванию). В dev-proxy имя мокается —
создавать заранее не требуется.

## Контроль размера билда

Бюджет ~20–25 МБ. Веб-экспорт Godot 4.7 = wasm-рантайм (~15–20 МБ сжатого) + `.pck`
с ассетами. Текущие ассеты проекта лёгкие (процедурные текстуры/звук, пиксель-арт
атласы — суммарно сотни КБ), так что основной вес — сам движок. Проверять размер
`build/web/*.wasm` + `*.pck` после каждой сборки; тяжёлые ассеты отмечать сразу.
Опции ужатия: без отладочного билда (release), gzip/brotli на хостинге Яндекса
(отдаётся автоматически).
